# Relatenta v1.0.0 — Feature Specification

**Version:** 1.0.0
**Date:** 2026-03-13
**Status:** Released

---

## 1. Product Overview

Relatenta is a Research Relationship Visualization platform that ingests scholarly publication data from OpenAlex API (250M+ records) or CSV files, and renders interactive 4-layer network graphs and heatmaps to reveal collaboration patterns across researchers, topics, organizations, and countries.

---

## 2. System Architecture

```
streamlit_app.py              (UI, sidebar, 4 tabs: Graph/Heatmaps/Reports/How-to)
  |
  +-- app/db.py               (In-memory SQLite, StaticPool, session isolation)
  +-- app/models.py           (13 SQLAlchemy ORM tables)
  +-- app/crud.py             (CRUD ops, edge computation, author merge)
  +-- app/connectors_openalex.py  (OpenAlex API client)
  +-- app/services_graph.py   (4-layer network graph builder)
  +-- app/services_heatmap.py (Author-keyword, Nation-nation heatmap)
  +-- app/services_export.py  (CSV/ZIP export)
  +-- app/services_report.py  (Analytics report generation)
```

### Database Schema (13 tables)

| Table | Purpose | Key Fields |
|-------|---------|------------|
| `authors` | Researcher entities | id, normalized_name, display_name, orcid |
| `author_aliases` | Name variations | author_id, raw_name, source, confidence |
| `organizations` | Institutions | id, name, country_code, city |
| `venues` | Journals/conferences | id, name, type, issn, publisher |
| `works` | Publications | id, doi, source_uid, title, abstract, year |
| `work_authors` | Work-Author junction | work_id, author_id, position |
| `work_affiliations` | Author-Org per work | work_id, author_id, org_id, country_code |
| `keywords` | Research topics | id, term_norm (unique), term_display |
| `work_keywords` | Work-Keyword junction | work_id, keyword_id, weight |
| `coauthor_edges` | Pre-computed co-authorship | a_id, b_id, weight, evidence_count |
| `org_edges` | Pre-computed org collaboration | org1_id, org2_id, weight |
| `nation_edges` | Pre-computed nation collaboration | n1, n2, weight |
| `merges` | Audit trail for author merges | entity_type, kept_id, removed_id |

## 3. Feature Details

### 3.1 Data Ingestion

**OpenAlex Search:**
- Search by name, ORCID, or Google Scholar URL
- Disambiguation via H-index, citation count, ORCID, research topics
- Configurable pagination (up to 600 papers per author)
- Automatic extraction of authors, affiliations, keywords, venues

**CSV Import:**
- Upload works, authors, affiliations, keywords via CSV
- ZIP restore for full data persistence

### 3.2 Network Visualization (4 Layers)

| Layer | Nodes | Edges | Edge Weight |
|-------|-------|-------|-------------|
| Co-authorship | Authors | Shared publications | # of joint papers |
| Keywords | Research topics | Co-occurrence in papers | # of co-occurrences |
| Institutional | Organizations | Joint publications | # of joint papers |
| National | Countries | International co-authorship | # of cross-border papers |

**Visualization Engine:** PyVis (vis.js wrapper)
- Force-directed layout with Barnes-Hut gravity
- Dynamic node sizing (connection count)
- Dynamic edge width (weight-proportional)
- Color coding by node type (blue=author, orange=keyword, green=org, purple=nation, red=focus)
- Interactive controls: drag/pan, scroll zoom, physics toggle (Space key)
- Auto-stop physics after 5 seconds

**Focus Mode:**
- Full Network (highlight): Show all nodes, highlight focus nodes in red
- Focus Only (isolate): Show only focus nodes and direct neighbors

### 3.3 Heatmap Analysis

- **Author-Keyword Matrix:** Top 30 authors x Top 30 keywords, Viridis color scale
- **Nation-Nation Matrix:** Symmetric collaboration intensity matrix

### 3.4 Analytic Reports

- Publication trend (papers per year bar chart)
- Top 20 authors by paper count
- Top 20 research keywords
- Country distribution (top 20)
- Top 15 collaborator pairs
- Top 15 keyword co-occurrence pairs
- Venue distribution (top 15)
- Highlight papers (top 15 by citation count)
- Simplified collaboration network graph

### 3.5 Data Management

- ZIP export (all tables as CSV)
- ZIP restore with automated edge recomputation
- Database reset
- Author merge with audit logging

## 5. Known Limitations (v1.0.0)

1. **Static relationship display** — No temporal evolution of networks
2. **No insight generation** — Visualizes data but doesn't interpret it
3. **No recommendation** — No collaborator/topic/paper suggestions
4. **No community detection** — Cannot auto-identify research clusters
5. **2D visualization only** — PyVis force-directed, limited for large networks
6. **No gap analysis** — Cannot identify unexplored research areas
7. **No trend detection** — Cannot identify emerging/declining topics
8. **Session-only persistence** — Data lost when browser closes (ZIP backup required)
